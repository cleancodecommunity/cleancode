What is Clean Code and Code Smell

Clean Code and Code Smell

Every digital machine (including our PCs and mobile phones) is run by software made of some codes. The functionality of these machines is defined in the Source-Code of the software, therefore, the main expectation from a software system is to work as expected. This expectation is what developers and customers care about the most. Like any other product, software has quality features and not all those features are revealed and realised easily when the software is launched. The lack of quality features in the software system has negative impact on the performance and maintenance cost in long term.

The term Clean Code was born in line with producing software systems (source-code) with quality features. In contrast to Clean Code, the term Code Smell was invented to indicate the lack of quality features or flaws in the source code. As the author of Clean Code book has stated, "Even bad code can function. But if code is not clean, it can bring a development organization to its knees". Therefore, a clean code or a smelly code, has nothing to do directly with software functioning. The impacts of code smell are revealed in the longer term.

The main impact of the code smell existence may be observed during the course of software maintenance. Software maintenance is an umbrella term that includes many different activities such as code debugging, code refactoring (code migration), update and code search. All these functionalities suffer from the existence of code smell. On average 75% of the TCO (Total Cost of Ownership) is spent on maintaining the software system. Therefore the existence of code smell has significant financial impact as well. Clean code has impact on the energy consumption. Two software systems that perform the exact same functionality with different source code design might have different energy consumption. This difference in energy consumption is mostly due to the existence of code smell. The energy consumption is usually referred to the amount of electricity energy that is consumed by the machine (CPU usage, memory RAM and etc).

Let's see what constitutes code smell. One of the coding practices that can lead to a code smell is the naming convention. Different functions and code artefacts in the software system have names. The common sense practice is to pick meaningful names for each artefact according to their functionalities and roles. This would potentially help make the process of source code debugging easier. The other example of code smell is to create too much of a dependency among source code artefacts. These dependencies would create a spider net where touching and updating one aspect would have an impact on many other aspects. Object oriented programming languages are the solution to this type of code smell only if they are properly applied. The other example would be the use of an inappropriate data structure to keep the data. The use of inappropriate data structure would potentially increase the running time of multiple tasks such as adding a new element, traversing and searching for an element, deleting an element and etc. Adding comments in the source code is a good habit that most developers have, however, there are good and bad practices in adding comments in the source code. Comments should always be related to the source code and provide information on the functionality of the source code and/or how the source code artefacts were crafted. This type of information would help new developers to understand the code for further development. Any other irrelevant types of comments would 1) increase the size of the source code unreasonably large and 2) might potentially mislead the new developers. Comments that are too obvious or too complicated are considered code smell. Commented code (these days we have control version systems so no need to keep the old version of the code as comment) is also considered as code smell.

There are several other types of code smell. Some of which are specific to different programming languages. Therefore, clean code is a multidimensional measure. To start we need to know and get familiar with code smells and analyse the system to detect their existence if there is any. The main goal of the Clean Code Community is to highlight the importance of code smell free software system and bring together with different discussions around different code smells.
